Runbook

Project: KAP myApps Portal  |  Last updated: 2026-04-03  |  Version: 1.0

1. Opening the app

The app requires no installation. There is no server to start.

  1. Open index.html in any modern web browser (Chrome, Edge, Firefox, Safari).
  2. Log in with your username and password.
  3. Click the app tile you want to use (VMS, HWMS, or Security).

Each module (myApps_VMS.html, myApps_HWMS.html, myApps_Security.html) can also be opened directly in a browser if you already know the URL.

The app requires an internet connection to reach Supabase. It will not work offline.

2. Development workflow

2.1 Making changes

  1. Open the relevant file in a code editor (VS Code recommended).
  2. Install the "Live Server" VS Code extension for auto-reload on save.
  3. Make your change and save — the browser refreshes automatically.
  4. Test in the browser before committing.
Note: myApps_HWMS.html (~1 MB) and myApps_VMS.html (~714 KB) are very large. Opening them in an editor may be slow. Search for the section you need before scrolling.

2.2 Shared code

Changes to myApps_Common.js affect all four apps. Test all apps after changing it.

3. Deployment

3.1 GitHub Pages (current method)

  1. Commit and push your changes to the main branch.
  2. GitHub Pages automatically serves the updated files.
  3. Wait 1–2 minutes for the deployment to propagate.
  4. Open the GitHub Pages URL to verify the change is live.

3.2 Manual deployment to internal server

  1. Copy all .html and .js files to the server's web root.
  2. The files can be served by IIS, nginx, Apache, or any static file server.
  3. No build step or dependency installation is required.

4. Database (Supabase)

4.1 Accessing the database

  • Log in at supabase.com/dashboard.
  • Select the KAP project (ehzfknwkerafblnibhps).
  • Use the Table Editor to view and edit data directly.
  • Use the SQL Editor for complex queries or migrations.

4.2 Backups

  • Supabase takes automatic daily backups (available in Dashboard → Settings → Database → Backups).
  • For manual backups: Dashboard → Settings → Database → Download backup.
  • Always take a manual backup before running any schema migration.

4.3 Schema changes

  • All schema changes (adding columns, tables, indexes) are made via the Supabase Dashboard or SQL Editor.
  • There is no migration framework — document any schema changes in the changelog.
  • After a schema change, update myApps_Common.js (SB_TABLES mapping) and any affected HTML files.

5. User management

  • Users are managed from within the portal (index.html) by an admin user.
  • To create a new user: log in as admin → go to Users tab → Add User.
  • To reset a password: edit the user record in the Users tab or directly in Supabase Dashboard → vms_users table.
  • To disable a user: deactivate the account in the Users tab.
