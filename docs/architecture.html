Project: KAP myApps Portal  |  Last updated: 2026-04-03  |  Version: 1.0

1. Overview

KAP myApps is a set of pure client-side web applications. There is no application server — all logic runs in the browser. The only backend is Supabase, a hosted PostgreSQL service accessed directly from the browser via HTTPS REST API calls.

  ┌─────────────────────────────────────────────────────────┐
  │                      User's Browser                      │
  │                                                           │
  │  index.html          Login portal + app launcher          │
  │  VMS.html     Vehicle Management System            │
  │  HWMS.html    Hardware/Warehouse Mgmt System       │
  │  Security.html  Security Surveillance              │
  │                                                           │
  │  Common.js    Shared: Supabase client, DB helpers, │
  │                       session, spinner, export utilities   │
  └──────────────────────────────┬──────────────────────────┘
                                  │ HTTPS (Supabase REST API)
                                  ▼
  ┌─────────────────────────────────────────────────────────┐
  │                    Supabase (hosted)                      │
  │   PostgreSQL database   │   Realtime subscriptions        │
  │   Row Level Security    │   Auth (not used — custom auth) │
  └─────────────────────────────────────────────────────────┘

4. Authentication flow

  User opens index.html
    → Login form shown (username + password + math CAPTCHA)
    → On submit: query vms_users table in Supabase
    → If credentials match: store session in memory (+ localStorage if "remember me")
    → Portal page shown with available app tiles
    → Clicking a tile opens the relevant HTML file
    → Each module re-authenticates from localStorage session on load

Note: Supabase Auth is not used. Authentication is custom, using the vms_users table directly. The Supabase anon key is public by design — Row Level Security (RLS) on Supabase enforces data access rules.

5. Shared module system (Common.js)

All HTML modules load Common.js via a <script> tag. The common file exposes:

  • Supabase client (_sb) and initialisation logic with CDN retry
  • Database helper functions for all CRUD operations
  • Global loading spinner (showSpinner() / hideSpinner())
  • Session management and current user object
  • Excel/CSV export utilities
  • Module hook points (_onPostBoot, _onRefreshViews, etc.) that each module overrides

6. Design system

All apps share a consistent visual language defined via CSS custom properties:

  • Accent colour: #2a9aa0 (teal)
  • Background: #f0f2f5
  • Surface: #ffffff
  • Text: #1e2329
  • Danger: #dc2626 (red)
  • Success: #16a34a (green)
  • Border radius: 12px
